# AI-Workflows

A curated collection of prompt templates designed to enhance AI-assisted coding workflows with GitHub Copilot. These templates provide structured guidance for various development scenarios, enabling more efficient and consistent pair programming between developers and AI.

## Overview

This repository contains specialized instruction templates (`.copilot-instructions.md` files) that serve as blueprints for AI-driven development tasks. Each template is crafted to optimize the interaction between human developers and AI assistants, focusing on specific domains such as frontend development, DevOps, testing, SEO, and more.

## Available Templates

- **Service Development**: Templates for creating frontend services with emphasis on user experience and functionality.
- **DevOps**: Instructions for containerization, deployment automation, and infrastructure setup.
- **Unit Testing**: Guidelines for implementing comprehensive test suites.
- **SEO Optimization**: Workflows for search engine optimization tasks.
- **Template Base**: A foundational template for creating new instruction sets.
- **Real Data Integration**: Instructions for transitioning from mock data to real data sources.

## How to Use

1. **Browse Templates**: Explore the available `.copilot-instructions.md` files to find a template that matches your development scenario.
2. **Customize**: Copy the relevant template and adapt it to your specific project needs, filling in placeholders and adjusting parameters as necessary.
3. **Integrate with Copilot**: Use the customized template as a prompt when working with GitHub Copilot or any other Agentic AI in your IDE.
4. **Iterate**: Refine the template based on your experiences to improve future interactions.

## Contributing

We welcome contributions from the community to expand and improve this collection of AI workflow templates. Here's how you can contribute:

### Guidelines

- **Template Quality**: Ensure templates are clear, comprehensive, and follow the established structure (Role, Mission, Goals, Instructions, Prerequisites, Tech Stack, Inputs, Outputs, Follow-up).
- **Relevance**: Focus on practical, real-world development scenarios that benefit from AI assistance.
- **Consistency**: Maintain consistency in formatting and terminology across templates.
- **Documentation**: Provide clear descriptions and examples within the template.
- **Testing**: Test templates in actual development scenarios to ensure they produce effective results.

### Steps to Contribute

1. **Fork the Repository**: Create a fork of this repository on GitHub.
2. **Create a New Branch**: Use a descriptive branch name (e.g., `add-template-[topic]`).
3. **Add Your Template**: Create a new `.copilot-instructions.md` file following the naming convention (e.g., `topic.copilot-instructions.md`).
4. **Update README**: If adding a new category, update the "Available Templates" section in this README.
5. **Submit a Pull Request**: Provide a clear description of your contribution and how it enhances the repository.
6. **Review Process**: Your PR will be reviewed for adherence to guidelines and overall quality.

### Code of Conduct

Please maintain a respectful and collaborative environment. Contributions should be constructive and focused on improving the repository.
